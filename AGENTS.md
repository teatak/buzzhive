# AGENTS.md

1. Admin UI 使用 shadcn 时，如果官方有组件，必须通过 `npx shadcn@latest add <component>` 引入官方组件，不手写替代组件。
2. Admin UI 需要扩展 shadcn 官方组件能力时,必须新增业务包裹组件承载扩展,禁止直接修改官方组件源码。
3. 修改 Admin UI 布局前必须先搞清现有样式/组件的职责和复用范围；能通过删除、拆分、重命名旧抽象解决时，优先做减法，禁止只在原有补丁上继续叠补丁。
4. 项目当前处于初期阶段，默认不需要兼容旧行为、旧配置、旧数据结构或旧接口；需要考虑将来的扩展，数据结构和接口一定要定义清楚，清晰，稳定。
5. Admin UI 新增功能前必须先做信息架构拆分：明确实体、页面边界、主列表、详情页、设置页和用户操作路径。
6. 禁止把多个一级实体的 CRUD 堆到同一个页面；除非用户明确要求做成总览页，否则不同一级实体必须拆成独立页面或详情页。
7. 列表页只展示当前实体的核心信息和高频操作；跨实体配置必须进入详情页、关联页或专门管理页。
8. 临时调试页必须明确标记为临时，不允许在临时页基础上直接扩展成正式功能。
9. Admin UI 实现前，如果页面同时出现 3 个以上核心实体，必须先停下来重新设计页面结构。
